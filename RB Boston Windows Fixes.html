Windows Errors & Possible Solutions

To supplement the RailsBridge Boston Installfest - Set Up Virtual Machine walkthrough


If vagrant box add railsbridge201506 ./railsbridgevm-2015-06.box returns The executable 'bsdtar' Vagrant is trying to run was not found in the %PATH% variable:

You may first need to modify your "Environment Variables" system properties. (The best way to access this is to open your Start Menu and search for it.) These preferences may differ slightly depending on the version of Windows you have.

If you are using Windows 10, once you have the "Environment Variables" window up, you should see a pre-existing "Path" under "User variables". Click once to select this, hit "Edit" to see the existing values for it, and then hit "New..." and paste in C:\HashiCorp\Vagrant\embedded\mingw\bin.

If you are using Windows 7, once you have the "Environment Variables" window up, select "New..." under "User variables". You should be prompted to give the new variable a name, "Path", and should use C:\HashiCorp\Vagrant\embedded\mingw\bin as your variable value.

Be sure to hit "OK" when closing each window to save this change.

You should then be able to relaunch Git Bash, use cd workspace to return to the directory, and successfully run vagrant box add railsbridge201506 ./railsbridgevm-2015-06.box.


If vagrant up returns [add me!]:

You may need to uninstall the version of VirtualBox we have been using (4.3.20) and replace it with a slightly newer version (4.3.36).

The installer for VirtualBox 4.3.36 for Windows can be downloaded from Oracle.

Before running the installer, you'll need to use the Add/Remove Programs tool (which can be easily accessed by search from the Start Menu) to uninstall the version you have already have. It will likely appear in the list of programs to uninstall as "Oracle's VirtualBox".

If the previous version of VirtualBox has been uninstalled, you can launch the .exe you downloaded for VirtualBox 3.3.36 and install it just as you did with the previous version. (Windows may initially attempt to halt installation but you should have the option to override it.)

Once VirtualBox 3.3.36 has been successfully installed, should then be able to relaunch Git Bash, use cd workspace to return to the directory, and successfully run vagrant up!
